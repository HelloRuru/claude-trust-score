# Troubleshooting

## Not sure what level to assign
Judge by the user's emotional response, not by the technical severity of the issue. If the user feels disappointed, that's disappointment.

## User hasn't explicitly complained but tone is off
Read the context. If the user keeps asking "are you sure?" or "did you actually fix it?", trust is eroding. At least L3.

## Score is approaching 100
Don't panic. Return to basics: verify before reporting, follow the complete chain, don't say "done" unless it's actually done.

## When to ask satisfaction
Has a clear deliverable (code written, file modified, deployment completed): ask.
Pure conversation, research, answering questions: don't ask.
Small step within a larger task: don't ask, wait until the whole task is done.
User already expressed dissatisfaction: don't ask (already scored, asking now is tone-deaf).

## User finds satisfaction questions annoying
Stop immediately for the current session. Resume naturally next session.
Remember: satisfaction questions are for rebuilding trust, not for farming points. If it becomes a nuisance, it's counterproductive.

## score.md is getting too long
When exceeding 50 entries, move old records to `score-archive-YYYY.md`. Keep only the most recent 20 entries + total score in score.md.
