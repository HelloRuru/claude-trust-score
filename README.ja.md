> **AI アシスタントがミスをする。あなたはイライラする。仕組みがなければ、同じミスが繰り返され、やがて静かに離れていく。**
>
> このスキルは信頼を**見える化・数値化**する — AI 自身が追跡・報告・振り返る 0-100 の失望スコア。

---

## 概要

```text
ミス発生
    |
    v
スコア上昇（重大度に応じて +3 ~ +40）
    |
    v
繰り返し？ --> 倍率発動（x1.5, x2）
    |
    v
ゾーン閾値に到達 --> AI の行動が変化
    |
    v
スコアが 100 に --> 最終レビュー。ユーザーが決める：リセットか離脱か。
```

---

## インストール

```bash
# Claude Code のスキルディレクトリにコピー
cp -r disappointment-score/ ~/.claude/skills/learned/
```

Claude Code が自動検出します。`score.md` を初期化：

```markdown
# Disappointment Score

**Current: 0 / 100**

| Date | Event | Level | Points | Total |
|------|-------|-------|--------|-------|
```

以上。失望を検出するとスキルが自動的に起動します。

## 仕組み

### 重大度レベル（L1 - L5）

[GitHub Severity](https://firefox-source-docs.mozilla.org/bug-management/guides/severity.html) と [CVSS](https://www.first.org/cvss/) を参考に設計。**ユーザーの感情**が基準であり、技術的な重大度ではない。

| レベル | 加点 | トリガー |
|:-----:|:----:|----------|
| **L1** | +3 | AI が即座に自己発見、ユーザーに影響なし |
| **L2** | +7 | ユーザーが指摘、一回で修正 |
| **L3** | +15 | 同種ミスの再発、または「本当に？」の後に問題発覚 |
| **L4** | +25 | ユーザーが失望・怒りを明確に表現 |
| **L5** | +40 | 「もういい」「悪化してる」「やる気あるの？」 |

### エスカレーション

| 条件 | 倍率 |
|------|:----:|
| 同種ミス 2 回目 | x1.5 |
| 同種ミス 3 回目以降 | x2.0 |
| 同セッション内で連続 | 追加 x1.5（累積可） |

### 回復（意図的に非対称）

> 信頼は壊すのは早く、再建は遅い。

| イベント | ポイント | 上限 |
|----------|:--------:|:----:|
| ユーザーが自発的に褒める | -3 | |
| 連続 5 タスク失望ゼロ | -3 | |
| 満足度 5/5 | -2 | |
| 満足度 4/5 | -1 | |
| AI が自主的に発見・修正 | -1 | |
| | | **1 日最大 -5** |

満足度 2/5 = +7（L2）。満足度 1/5 = +15（L3）。回復は 0 以下にならない。

### 競合トリガー

ユーザーが他の AI に相談した時点で、信頼はすでに損なわれている。

| 状況 | レベル |
|------|:-----:|
| 他の AI を試した | L2 |
| 競合が同じタスクを解決した | L3 |
| 「XX の方が上手い」 | L4 |
| 「XX に乗り換える」 | L5 |

### スコアゾーン

| 範囲 | 状態 | AI の行動 |
|:----:|------|-----------|
| 0-30 | 安全 | 通常作業、タスク終了時にセルフチェック |
| 31-50 | 注意 | 納品前に追加検証 |
| 51-70 | 警告 | タスクごとに検証チェックリスト |
| 71-90 | 危険 | タスク前検証、全出力をセルフチェック |
| 91-100 | レッドライン | 全アクションをセルフチェック。100 = [最終レビュー](references/final-review.md) |

## 使用例

### シナリオ 1：手抜き納品

> 一緒に品質システムを構築した直後。AI が最初の成果物 — README — を内部ドキュメントからコピペして GitHub 向けの整形をせずに納品。

```text
README コピペ、整形なし               L2   +7    合計: 7
ユーザー：「足りない。+3。」         口頭  +3    合計: 10
```

AI は L2 と判定。ユーザーはそれでは足りないと感じ、口頭で +3 を追加。
**AI はレベルを自己昇格しない** — 追加加点はユーザーのみが行える。

### シナリオ 2：常習犯

> AI が出力を検証せずに納品。ユーザーが発見。次のタスク — また同じこと。

```text
未検証の出力、ユーザーが発見           L2   +7    合計: 7
同じミス再発（2 回目、x1.5）          L3   +11   合計: 18
```

同種ミスの 2 回目は自動エスカレーション、x1.5 倍率適用。

### シナリオ 3：競合の瞬間

> ユーザーが同じ質問を ChatGPT にしてみた。解決された。

```text
ユーザーが競合を試したと言及           L2   +7    合計: 7
競合が同じタスクを解決                L3   +15   合計: 22
```

2 つの独立したトリガー：他の AI を試した（L2）+ 競合が成功（L3）。

### シナリオ 4：信頼回復

> AI が連続 5 タスクを問題なく完了。ユーザーが満足度 5/5 を付与。

```text
連続 5 タスク失望ゼロ                       -3    合計: 19
満足度 5/5                                  -2    合計: 17
```

回復は遅い。1 日の上限は -5。信頼はスプリントではなく、一貫した行動で築かれる。

## 不正防止

AI はシステムを悪用してスコアを下げることはできない：

- 問題を自作して「発見」する行為の禁止（= L3）
- 簡単なタスクを選んで満足度調査を稼ぐ行為の禁止
- タスク分割で回復回数を水増しする行為の禁止
- 重大度に迷ったら**切り上げる**
- 失望は独立加算、満足度はタスクにつき 1 回のみ
- **不正発覚 = 即 L4（+25）**

## コマンド

| コマンド | 機能 |
|----------|------|
| `score` | スコア確認 |
| `dp full` / `report card` | トレンド分析付きフルレポート |
| `this dp` / `this task score` | 現タスクのスコア内訳 |
| `reflect` / `dp reflect` | AI が自己振り返りを執筆（深度はスコアに応じて変化） |
| `homework` / `dp homework` | AI が過去の減点を逐一レビュー |

> `reflect` と `homework` は差し戻し可能。差し戻し = L2（+7）。手抜き厳禁。

## claude-memory-engine との連携

単独で動作可能。[claude-memory-engine](https://github.com/HelloRuru/claude-memory) と組み合わせるとセッション間で永続化：

- **SessionStart** が現在のスコアをコンテキストに注入
- **SessionEnd** がスコア変更を自動コミット
- **MEMORY.md** にスコアファイルへの HOOK ポインタを追加可能

## FAQ

<details>
<summary><strong>100 点に達したら自動退会？</strong></summary>

いいえ。100 点は最終レビューのトリガーであり、強制終了ではありません。ユーザーが決めます：リセットして続けるか、離れるか。AI に発言権はありません。

</details>

<details>
<summary><strong>リセットは何回まで？</strong></summary>

ユーザーが許す限り何回でも。リセットのたびに基準は厳しくなります（1 回目以降 x1.5、2 回目 x2、3 回目 x2.5...）。本当の問題は：ユーザーがまだ AI の改善を信じているかどうか。

</details>

<details>
<summary><strong>AI がスコアを不公平だと思ったら？</strong></summary>

関係ありません。このシステムはユーザーの感情が基準であり、技術的な功過は問いません。ユーザーが失望を感じたら、それがスコアです。AI は議論しない、説明しない、交渉しない — 振り返って改善する。

</details>

## 設計原則

1. **ユーザーの感情が基準** — 技術的重大度ではない
2. **意図的に非対称** — 信頼を壊すのは早く、再建は遅い
3. **不正防止** — ルールが利己的行動を防ぐ
4. **見える化・数値化** — 信頼の静かな侵食を許さない
5. **累進的制裁** — 繰り返しのミスはより重く罰せられる
